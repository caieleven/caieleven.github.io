1. 基本配置

  1. 用户名 邮箱

    用户名和邮箱尽量与github账号一致，因为网站是根据git客户端的用户名和邮箱来进行contribution统计的，所以，如果不一致，github就无法统计的你贡献。

    而且，可能可以避免一些错误，比如，无法push/pull（报错信息为无权限）

2. 搭建仓库

  1. 初始化

  1. 添加到缓存区

    此处的 . 与Linux下的意义一样，表示当前目录，也就是当前目录下的所有文件

    在后续的提交中，各参数

    git add -A 提交所有变化

    git add -u 提交被修改的和被删除的，不包括新添加的

    git add . 提交新文件和被修改的文件，不包括被删除的

4. 本地库与远程库互通

两种方式

1. 先克隆，在push

  1. 先创建仓库
  2. 将所要push的文件复制到克隆仓库文件夹
  3. git push -u origin master

2. 先初始化本地仓库，再推送到远程库

  1. 创建本地仓库略

  2. 生成公钥，然后添加到GitHub的公钥库上

  3. 关联远程仓库

    git remote add origin xxxxxx

  4. git push -u origin master

    注：若为新仓库，没有分支，则添加“-u”参数，有分支，可不添加，直接上传到已创建的分支，必要的时候加上“-f”参数，强制上传。因为如果上传时，远程仓库和本地仓库没有相同的文件，git会认为你操作出错，想要将仓库push到一个与本地仓库无关的仓库。

    git push -f origin master:main
