MakeFile工作流程

执行make命令时，make会在当前文件夹下找我们书写的编译规则，即Makefile文件。我们编写Makefile时，文件名最好写成”Makefile“，一般工程都这么写。

举个Makefile的例子：

在我们编译项目文件的时候，默认情况下，make 执行的是 Makefile 中的第一规则（Makefile 中出现的第一个依赖关系），此规则的目标称之为“最终目标”。

开始处理第一个规则，在执行这个规则所定义的命令之前，需要处理目标”main“的所有依赖文件的规则。

在处理这些规则时，有三种情况：

  1. 目标不存在，使用其规则创建它
  2. 目标存在，但是文件的任何一个依赖被修改，则重新生成它。
  3. 目标存在，所所有依赖均未被修改，不执行它所对应的命令。

这种分情况处理规则，很大程度上节省里项目构建时间。

Make file目标文件搜索

需要的文件存于不同的路径下，就需要使用Makefile的目录搜索文件功能。

常见的搜索的方法的主要有两种：一般搜索VPATH和选择搜索vpath。乍一看只是大小写的区别，其实两者在本质上也是不同的。

VPATH 和 vpath 的区别：VPATH 是变量，更具体的说是环境变量，Makefile 中的一种特殊变量，使用时需要指定文件的路径；vpath 是关键字，按照模式搜索，也可以说成是选择搜索。搜索的时候不仅需要加上文件的路径，还需要加上相应限制的条件。

Posted on 2021-08-02 Edited on 2021-08-11 In c/c++

cin简介

​ cin是c++标准输入流对象，即istream类的对象。同理，cout是ostream类的对象。

​ 当我们从键盘中输入字符串的时候，需要敲回车键才能将这个字符串输送到缓冲区中，然后敲入的这个回车键(\r)会被转换为一个换行符(\n)。

​ cin读取数据时，也是从缓冲区中获取数据，缓冲区为空时，cin的成员函数阻塞，等待数据的到来，一旦缓冲区中有数据，就触发cin的成员函数去读取数据。

cin常用输入方法

cin >>

​ cin可以连续从键盘读取想要的数据，以空白字符作为i分隔符。

​ 当cin>>从缓冲区读取数据时，若缓冲区第一个字符是空白字符（比如空格、tab、换行等），cin会将其忽略并清楚，读取下一个字符，若缓冲区为空，则继续等待。

​ 若略过空白字符，也有方法，此处不展开。

cin.get()

  1. 读取一个字符

    1. 第一种用法的返回值是一个int值，成功则返回读取字符的ASCII码值
    2. 第二个的返回值为一个cin对象，因此支持链式操作，比如： cin.get(a).get(b)

  2. 读取一行

cin.getline()

​ 函数原型

两种方式

  2. 生成公钥，然后添加到GitHub的公钥库上

  3. 关联远程仓库

    git remote add origin xxxxxx

  4. git push -u origin master
