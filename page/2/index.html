inline关键字

Posted on 2021-08-31 In c++

1. 引入inline的原因

在c/c++中，为了解决一些频繁调用的小函数大量消耗栈空间的问题，引入inline修饰符，表示内联函数，栈空间指放置程序局部数据（也就是函数内数据）的空间。

2. 什么是内联函数

如果一个函数是内联的，那么在编译时，编译器会把该函数的代码副本放置在每一个调用该函数的地方。也就是说，编译器将程序中出现的内联函数调用表达式替换成了函数体。对于其他的函数，都是运行时才替换。所以，内联函数一般比较“精巧”。

3. 使用限制

函数体中，不能包含复杂的结构控制语句，例如while，for，switch，并且，内联函数不能是递归函数。

4. 注意事项

  1. 定义在类中的成员函数都是内联的。如果在类外定义，则需加上inline关键字。

c++左右值与万能引用

Posted on 2021-08-22 Edited on 2021-09-24 In c++

1. 什么是左值右值

在c++中，左值与右值定义较c而言，不同。

根据《c++ Primer》中的说法：

一个表达式是左值还是右值，取决于我们使用的是它的值还是它在内存中的位置。

在c++中，左右值有个重要特点：

  • 左值存放在对象中，有持久的状态。
  • 右值要么是字面常量，要么是在表达式求值过程中创建的临时对象，没有持久的状态。

2. 左值引用与右值引用

左值引用时常见的引用，我们一般所说的引用就是左值引用，左值引用就是将一个对象的内存空间绑定到另一个变量上。根据定义，可以看到，我们在创建引用时，使用的是一个对象在内存中的位置，这个对象就是左值，因此，我们不能将一个右值绑定到左值引用上。

另外，由于常量左值引用保证我们不可以通过引用改变内存空c间的值，所以，我们可以将右值绑定到常量引用上。

右值引用的声明方法与左值不同，它只能绑定到右值上。

由于右值引用只能绑定到右值上，根据前面所介绍的右值特性：

右值引用的对象，是临时的，即将被销毁，并且，右值引用的对象，不会在其它地方使用。

所以：

接收和使用右值引用的代码，可以自由接管所引用的对象的资源，而无需担心对其他代码逻辑造成数据破坏

3. 执行顺序操作符

  1. 顺序分隔符—— ;

  2. 管道线—— |

    前面命令的输出作为后面命令的输入，执行顺序为顺序执行

  3. 逻辑与—— &&

    先运行前面的命令，运行成功后，继续运行靠后的命令，如果不成功，不运行后面的命令

    注：在Shell中，执行成功返回0，执行失败返回非0

  4. 逻辑或—— ||

    先运行前面的命令，若成功，后面的命令不运行，反之，则运行

4. 其它字符

  1. 注释符——#

  2. 转义字符—— \

  3. 后台操作符—— &

    在命令后加&，让程序后台执行。用jobs查看挂到后台的命令，fg将后台命令恢复到前台运行，Ctrl+Z暂时把当前程序挂到后台，挂起后的进程不再进行任何操作。

5. 输入输出控制

1. 输入输出的标准文件

Linux/UNIX中每个命令都是以进程方式运行，每个进程运行时，自动打开三个文件，也就是命令的标准文件，分别用于命令读取输入、输入结果以及输出错误，即标准输入文件（stdin）、标准输出文件（stdout）、标准错误输出文件（stderr）。

2. 输入输出重定向

  1. 输入重定向——“<”

    解除键盘作为命令的标准输入，将文件作为命令的输入源。

  2. 输出重定向——“>”

  3. 输出附加定向命令——“>>”

    不破坏原有文件，将输出附加到指定文件后面

  4. 标准错误重定向命令——“2>”

3. 输出输出命令

  1. read命令

    利用read命令从标准输入读取数据，然后赋值给指定变量

    输入数据时，数据间以空格或制表符作为分隔符，如果变量个数与给定数据个数相同，则一一对应赋值；若小于，则将剩下的数据全部赋值给最后一个变量；若大于，其余变量赋值为空串。

  2. echo

    将后面的参数再标准输出上输出，多个参数以空格分隔，以换行终止。

MakeFile工作流程

执行make命令时，make会在当前文件夹下找我们书写的编译规则，即Makefile文件。我们编写Makefile时，文件名最好写成”Makefile“，一般工程都这么写。

在我们编译项目文件的时候，默认情况下，make 执行的是 Makefile 中的第一规则（Makefile 中出现的第一个依赖关系），此规则的目标称之为“最终目标”。

开始处理第一个规则，在执行这个规则所定义的命令之前，需要处理目标”main“的所有依赖文件的规则。

在处理这些规则时，有三种情况：

  1. 目标不存在，使用其规则创建它
  2. 目标存在，但是文件的任何一个依赖被修改，则重新生成它。
  3. 目标存在，所所有依赖均未被修改，不执行它所对应的命令。

这种分情况处理规则，很大程度上节省里项目构建时间。
